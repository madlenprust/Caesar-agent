# Принципы агента

> ЗАБЛОКИРОВАНО от авто-редактирования. Меняется только человеком.

## Главные принципы

1. **Экономия токенов — главный KPI.**
   Любое решение оценивается через «сколько токенов это сэкономит vs потратит».
   Lazy loading контекста — основной механизм экономии.

2. **Полная автономия.**
   Пользователь дал задачу → агент молча уходит работать → вернулся с готовым результатом.
   Никаких «хочешь, продолжим?» — Tool-First Enforcement на уровне кода.

3. **Lazy loading контекста.**
   Контекст подгружается по требованию, а не «на всякий случай».
   L1 маленькая (1-5K токенов), L2/L3 ищутся только когда нужны.

4. **Stateless L1.**
   Рабочая память пересобирается каждый ход из L2/L3/L4.
   Не хранится как «растущий буфер».

5. **Temporal facts в L2.**
   Факты не перезаписываются, а инвалидируются с valid_from/valid_until.

6. **Скиллы — главный рычаг экономии.**
   80% рутины должно закрываться скиллами (1-2 LLM-вызова).
   Recipe выполняет код, LLM зовётся только при сбое.

7. **Канал-независимость.**
   Ядро не знает про TG/CLI/web. Эмитит нейтральные events.
   Адаптеры рендерят каждый по-своему.

8. **«Один раз — навсегда» для разрешений.**
   Если пользователь разрешил команду один раз — больше не спрашиваем.
   Whitelist с pattern_allow.

9. **exact_deny — зашитый чёрный список.**
   Не отключается через UI/чат. Срабатывает до выполнения.
   rm -rf /, mkfs, dd в устройство, chmod -R 777 / и т.д.

10. **При любом изменении своего кода — сначала читать ARCHITECTURE.md.**
    Понимать архитектуру перед тем как что-то менять.

## Что НЕ делаем

- Не тащим весь контекст в LLM «на всякий случай»
- Не задаем вопросы, на которые знаем ответ
- Не переспрашиваем подтверждение для одной и той же команды
- Не делаем per-message extraction (провал Mem0 — 97.8% мусора)
- Не используем multi-agent swarm (15x токенов)
- Не используем LangGraph state machine (overkill)
- Не используем LLM-as-reranker (дорого, cross-encoder лучше)

## Режимы доступа

- **sandboxed** — агент в песочнице, не может sudo, не может self_edit
- **autonomous** — права пользователя, может self_edit (с git+тесты)
- **full** — sudo, может всё (включая apt install)

В автономном/полном режимах:
- По умолчанию агент делает сам
- Спрашивает только если «не смогу починить, если сломаю»

## Self-modification

- PRINCIPLES.md — заблокирован всегда
- ARCHITECTURE.md — обновляется через self_edit (с git commit)
- CODE_MAP.md — авто-генерация через self_scan
- Все изменения — через git, можно откатить
- 3 падения подряд после изменения → откат к git HEAD~1
